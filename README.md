# Queue Management Telegram Bot

Асинхронный Telegram-бот для автоматизации управления электронными очередями. Архитектура проекта спроектирована с упором на отказоустойчивость, высокую производительность СУБД при конкурентных запросах и гибкое разграничение прав доступа.

Этот репозиторий демонстрирует базовую архитектуру, паттерны проектирования и стек технологий, которые использовались при разработке аналогичного коммерческого сервиса психологического тестирования (находящегося под NDA).

## 🛠 Технологический стек

* **Language:** Python 3.10+
* **Framework:** python-telegram-bot (Asyncio)
* **Database:** PostgreSQL
* **DB Driver:** asyncpg (полностью асинхронное взаимодействие с базой данных без блокировки основного потока)

## 🏗 Архитектурные особенности и паттерны

1. **Non-blocking Async Архитектура:** Весь ввод-вывод (I/O), включая запросы к API Telegram и транзакции к PostgreSQL, реализован в асинхронном режиме. Это позволяет боту обрабатывать сотни конкурентных запросов от пользователей одновременно без задержек.
2. **Оптимизация работы с БД:** Использование пула соединений в `asyncpg`, параметризованные SQL-запросы для защиты от SQL-инъекций и оптимизированные индексы для быстрого поиска связей «пользователь-реферал».
4. **Административная панель:** Встроенная ролевая модель доступа. Администраторы имеют доступ к интерактивным командам для управления очередями, выгрузки логов и изменения параметров тестирования в реальном времени.
